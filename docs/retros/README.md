# Phase Retrospectives

Per-phase retrospectives. One file per phase: `phase-NN.md`.

A retrospective is required at the end of every phase per CONSTITUTION
Priority 7. The phase-boundary-auditor (CLAUDE.md Rule 24) verifies the
retro file exists before PR creation on standard phases.

Each retro should answer:

1. **What went well** — concrete things to repeat
2. **What was challenging** — concrete obstacles
3. **What to improve** — concrete actions for the next phase
4. **Advisories raised** — log every BLOCKER/FINDING/ADVISORY, with
   target task and resolution path
5. **Rules that prevented or caught failure** — track rule efficacy for
   the sunset evaluation (CLAUDE.md Rule 15)

The summary line for each phase is added to `docs/RETRO_LOG.md`.
